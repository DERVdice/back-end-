Для проверки запустить cmd в текущей директории и выполнить следующие комманды:

1. Загрузка image в docker из .tar архива:
docker load < derv-dice.latest.tar

2. Запуск контейнера с именем travolta на 8080 порте. Если 8080 занят, то заменить первое число на свободный порт
docker run --name travolta -i -t -p 8080:8080 derv-dice:latest

3. Запуск веб-сервера в контейнере в фоновом режиме
go run WebServer.go &

(Все, что он умеет делать - отправлять домашнюю страницу index.html по адресу "/" или travolta.html по адресу "/travolta"

4. Открыть браузер по адресу http://192.168.99.100:8080/ и увидеть (я надеюсь) index.html

У меня все работает именно по этому адресу т.к. я "счастливый обладатель" windows 10 home, которая не поддерживает 
docker desktop, поэтому я пользовался docker toolbox, работающей на основе oracle virtualbox, которая в свою очередь,
дает именно такой стандартный ip docker'у

Если по адресу ничего нет, то проверить localhost:8080

Если не работает и он, то нужно узнать <docker-machine ip default>, вписав эту комманду в cmd, затем 
проверить его на порту 8080
